How this blog works

02/12/2020

Hello, my name is Adam Richardson and on this first post of my blog I wanted to discuss how it was put together. This site is a static site made using a variety of tools but most importantly Go Web Templates and TOML markup. My goal was to build a minimal blog that allows me to easily make posts.

Static site?

- No worry of DDOS - Easily parsable by robots - CDN hostable - Fast

Why not use an existing static site generator?

- Its easy to make your own - Building your lets you completely customize your site - Customize the process of your site - Free from keeping up with the breakneck pace of web tools

Go Templates

- Built into the go standard library - Easy and robust - Uses go structs as the model - example rendering home.toml in home.tmpl and rss.tmpl - htmlSafe and FuncMap for creating template extensions

TOML

- Markup language easily edited by humans - Can be serialized into a go struct

Organization

- tools folder - main.go - assets - templates - data - Makefile

How well does it work?

- DRY! - easy to deploy - updating the models, templates and data not centralized
